## 状态

```bash
mailrelay status
```

输出 Queue 深度、待发送回复、死信数量、最后轮询、运行错误、Catalog Hash 和最近执行。

## 死信重放

修复外部依赖后显式重放，不自动无限重试：

```bash
mailrelay replay queue 42
mailrelay replay reply 17
```

Queue 将外部依赖故障视为可重试错误，并按有上限的退避策略重新调度。配置缺失、参数非法或策略拒绝属于终止错误，会直接进入 dead 状态。先用 `mailrelay status` 确认失败分类，修复根因，再执行 replay。

## 72 小时验收

```bash
mailrelay doctor
mailrelay soak --duration 72h
```

验收期间测试合法邮件、未授权邮件、重复 Message-ID，以及 IMAP、SMTP、HTTP 短暂中断。通过标准：零未授权执行、零重复执行，并且结束时没有 pending/dead Queue 或 Reply。

## 备份

停止 MailRelay 后同时备份 YAML 和 SQLite 数据库，或使用 SQLite 在线备份工具。
