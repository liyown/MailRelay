# MailRelay v0.1 Reliability Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the single-host MailRelay golden path durable, observable, security-tested, and releaseable as v0.1.

**Architecture:** SQLite outbox and dead-letter transitions separate command execution from delivery. Existing adapters remain behind interfaces so local protocol fixtures can test retries and reconnect behavior; CLI exposes recovery and soak operations without widening the runtime command model.

**Tech Stack:** Go 1.25+, SQLite, local TLS test servers, GitHub Actions, GoReleaser.
